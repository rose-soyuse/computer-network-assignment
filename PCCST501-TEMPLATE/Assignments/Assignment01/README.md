# Assignment 1

## Title
How the Internet Delivers a Web Page
## Objective
The website Google (www.google.com) was used to study how a web page is delivered over the Internet. The communication process was explained using the TCP/IP protocol stack, where DNS translates the domain name into an IP address, TCP establishes a reliable connection, and HTTP requests and receives the webpage. The assignment also explains the client–server architecture, compares HTTP and FTP, describes the role of cookies, and illustrates the complete communication flow between the browser and the web server.
## Solution
The study demonstrates the complete sequence of events that occurs when a user enters a website URL in a browser. It shows how DNS resolves the domain name, how HTTP and TCP work together to retrieve the webpage, and how the browser finally displays the requested web page. It also highlights the importance of cookies and the client–server model in modern web communication.
## Output
The study demonstrates the complete sequence of events that occurs when a user enters a website URL in a browser. It shows how DNS resolves the domain name, how HTTP and TCP work together to retrieve the webpage, and how the browser finally displays the requested web page. It also highlights the importance of cookies and the client–server model in modern web communication.
## Conclusion
This assignment helped in understanding how the Internet delivers a web page using the TCP/IP protocol suite. It explains the roles of DNS, HTTP, TCP, and cookies in enabling reliable and efficient communication between the browser and the web server. The study shows that application-layer protocols are essential for accessing websites and other Internet services, making everyday online communication fast, reliable, and user-friendly.
